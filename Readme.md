# 🎵 K-Music Share (한국 음악 추천 및 공유 플랫폼)

본 프로젝트는 사용자가 좋아하는 한국 음악(K-Pop, 인디, 힙합 등)을 소개하고, 다른 이용자들과 실시간으로 추천 리스트를 공유하는 커뮤니티 플랫폼입니다.

## 🚀 주요 기능 (Key Features)

1. **사용자 인증**: 회원가입, 로그인 및 프로필 관리.
2. **음악 포스팅**: 곡 제목, 아티스트, 앨범 아트, 추천 이유 및 청취 링크 함께 공유.
3. **카테고리 분류**: 장르별(아이돌, 발라드, 트로트 등) 및 상황별(드라이브, 공부할 때 등) 태그 제공.
4. **커뮤니티 인터랙션**: 게시글 좋아요, 댓글 창, 팔로우 기능.
5. **검색 및 필터링**: 최신순, 인기순 필터 및 아티스트/곡명/태그 검색.
6. **플레이리스트 생성**: 개별 사용자가 추천 곡을 모아 자신만의 플레이리스트 구성.

## 🛠 기술 스택 (Tech Stack)

* **Frontend**: React.js / Next.js (SEO 최적화)
* **Backend**: Node.js (Express)
* **Database**: SQLite / MySQL (관계형 데이터 모델)
* **Authentication**: JWT 기반 (JSON Web Token)
* **Styling**: Tailwind CSS

## 📅 작업 방법 / 순서 (Workflow)

1.  **기획 및 요구사항 정의**: 구체적인 기능 명세서 및 와이어프레임 작성.
2.  **데이터베이스 설계**: 사용자, 게시글, 댓글, 좋아요에 관한 엔티티 관계도(ERD) 구축 및 스키마 작성.
3.  **백엔드 API 개발**: 인증/인가 처리를 포함한 RESTful API (Auth, Post, Comment 등) 라우터 구축.
4.  **프론트엔드 UI/UX 구현**: React/Next.js 기반의 컴포넌트 단위 개발 및 백엔드 API와의 연동.
5.  **테스트 및 디버깅**: 로컬 테스트 및 다양한 엣지 케이스 점검.
6.  **배포**: Vercel(Front), Render 또는 AWS(Back)를 활용한 실서비스 런칭.

## 🗄️ Database 구조

데이터베이스 스키마는 `schema.sql` 파일에 정의되어 있으며, 주요 테이블은 아래와 같습니다:
- **Users**: 사용자 계정 및 프로필
- **Music_Posts**: 추천 음악 게시물 데이터
- **Comments**: 포스트의 댓글 정보
- **Likes**: 게시글 좋아요 데이터 (사용자별 중복 제한)

### 💡 팁
- 스포티파이(Spotify) Web API 또는 유튜브(YouTube) Data API 연동을 통해 수동 정보 입력 없이, 곡 제목 검색만으로 앨범 아트와 곡 정보를 자동 연동시킬 수 있습니다.
